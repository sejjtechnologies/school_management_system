Quick Start (for evaluators)

Fast steps to get the system running in a development/demo environment.

  1. Clone the repository and copy a `.env` file with configuration (DATABASE_URL, SECRET_KEY).
  2. Create the Postgres DB, then run migrations: alembic upgrade heads.
  3. Seed default users (see insert_default_users.py) then start the server: python app.py.
cd project-root
python -m venv .venv; .venv\Scripts\Activate.ps1
pip install -r requirements.txt
set -Path_to_.env
alembic upgrade heads
python app.py

Roles — Overview & Primary Responsibilities

Below are the primary user roles and their typical day‑to‑day tasks. Use this as a guide when training staff.

Admin

Full system administrative control.

  • Create and manage users, set roles and permissions.
  • Run data seeders and emergency fixes.
  • Manage integrations and system settings (email, payments sandbox).
Headteacher

Academic oversight and reporting.

  • Mark and review attendance, approve locked days.
  • Generate attendance summaries (daily/termly) and export CSV.
  • View staff assignments and approve timetable changes.
Teacher

Class-level operations.

  • Record daily attendance for assigned classes.
  • Enter marks and upload class notes or reports.
  • Access pupil lists and print class reports.
Secretary

Administrative data entry and records.

  • Register pupils, assign to classes/streams and maintain personal records.
  • Help parents with initial onboarding and data corrections.
  • Run bulk operations (imports) and reconcile records with paper ledgers.
Bursar

Finance and reconciliations.

  • Record payments, issue receipts, maintain payment log.
  • Download and reconcile daily receipts with bank statements.
  • Manage salary data and run salary-related reports.
Parent / Guardian

View-only / limited actions.

  • View child's attendance and reports, download receipts.
  • Update emergency contacts (where allowed) and view invoices.

Daily & Monthly Operational Checklist

Suggested running order for each working day and month-end tasks.

Daily (Headteacher / Teacher)
  • Check dashboard KPIs (attendance today, absent counts).
  • Teachers submit attendance by start of lessons; Headteacher reviews and locks day if complete.
  • Bursar records payments and uploads receipts; send digital copies to parents.
Monthly / Termly
  • Generate attendance summaries & export CSV for archive.
  • Run financial reconciliation and export ledgers for accountant.
  • Run backups before major migrations or upgrades.

Deploy & Run (production tips)

Production recommendations for reliability and security.

  • Use Gunicorn behind Nginx for Python WSGI hosting; set up systemd for process supervision.
  • Store secrets in environment variables; never commit `.env` to source control.
  • Enable TLS at the proxy (Nginx/Load Balancer) and protect admin routes behind IP or VPN if needed.
# example gunicorn start
gunicorn -w 4 -b 0.0.0.0:8000 app:app --reload

Database & Migrations

Key commands and tips for schema management.

  • Check current DB revision: alembic current.
  • List heads: alembic heads. If multiple heads exist, reconcile by stamping or merging.
  • Apply migrations: alembic upgrade heads. If a migration fails due to already-applied changes, use alembic stamp <rev>.
Tip: In development we sometimes enlarge alembic_version.version_num if revision names exceed the column width.

Diagnostics & Performance

How to investigate slow queries or long-running summary operations.

  1. Use the aggregate endpoint for attendance summaries to avoid many per-day requests.
  2. Hit the debug EXPLAIN route (development only) to get EXPLAIN ANALYZE output and inspect for sequential scans.
  3. If necessary, add composite indexes or create materialized views for heavy reports.

Sales & Onboarding Notes

How to present the system to potential customers and onboard them quickly.

  • Start with the business problem: show how hours saved in admin work translate to impact.
  • Demo the role flows: Admin -> Secretary -> Teacher -> Headteacher -> Parent, showing the same pupil flowing through each stage.
  • Offer a 2-week pilot (seed data, train 2 staff members, collect feedback) and include remote support in the proposal.
Demo idea: Walk through registering a pupil, marking attendance for a week, recording a payment, and generating a monthly attendance CSV — this showcases the full loop.

Support, Training & Documentation

Recommended support bundle for purchasers.

  • Starter training: 2-hour hands-on workshop for Admin & Secretary users.
  • Monthly remote support for the first pilot period (bug fixes, small changes).
  • Provide a PDF quick-start for non-technical staff and record a short walkthrough video.
